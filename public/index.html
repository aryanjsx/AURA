File Operations

Create, delete, rename, move, and search files anywhere on your machine. Smart keywords like desktop/ and ~/ resolve to real OS directories.

Monitor CPU and RAM, list running processes, check system health (Python, Git, Node, Docker, npm), and run allowlisted shell commands.

Project Scaffolding

Spin up a new project skeleton anywhere with create project — generates src/, tests/, README, .gitignore, and requirements.txt.

📋

Log Inspection

Tail any log file without leaving the CLI. show logs aura.log 10 gives you the last 10 lines instantly.

🔒

Security Built In

Protected system paths are blocked, shell commands use an allowlist, and every action is hash-chained to a tamper-evident audit log.

Developer-First CLI

Interactive REPL with banner, mode indicator, built-in help, and single-command mode via --yes. Clean errors, no tracebacks.

Architecture

An immutable core. Infinite plugins.

The core never changes. Every capability — voice, vision, memory, automation — is a plugin that hot-loads around it.

🧬 Modular Core + Plugin System

A tiny, battle-tested runtime orchestrates events, state, and the plugin lifecycle.

🔒 Core Is Immutable

Plugins cannot patch the core. Guarantees stability, security, and predictable upgrades.

🧩 Plugins Add Features

Drop a folder, register a manifest, ship a capability. Voice, vision, dev tools — all plugins.

aura/ 
AURA/
├── aura/
│   ├── cli.py           # CLI entry (python -m aura)
│   ├── core/            # Primitives: event bus, config, intent, schema
│   ├── runtime/         # Router, registry, engine, worker client, planner
│   ├── security/        # Sandbox, policy, safety gate, audit, manifest
│   ├── worker/          # Isolated execution subprocess
│   └── intents/         # Main-process text → Intent parsers
├── plugins/             # Worker-only plugin executors
├── docs/                # Architecture + phase plans
├── tests/               # pytest suite (214 tests)
└── plugins_manifest.yaml  # Authoritative plugin safety manifest
